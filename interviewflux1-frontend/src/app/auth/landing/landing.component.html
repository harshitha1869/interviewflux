WHY INTERVIEWFLUX

Enterprise-grade booking
built for scale

JWT Authentication

Stateless, secure Bearer token auth on every protected API call. Tokens are auto-attached via Angular HTTP Interceptor across all requests.

POST /auth/login

3-Tier Role System

Super Admin, Admin, and User — each with dedicated dashboards, route guards, and scoped API access.

SUPER_ADMIN ADMIN USER

Distributed Backend

3 Spring Boot instances behind NGINX on port 9090 for high availability and load distribution.

Pessimistic Locking

Database-level locking prevents double-booking race conditions in concurrent slot booking scenarios.

Real-time Feedback

Loading spinners, success/error toast notifications, and disabled states for already-booked slots.

HOW IT WORKS

Three roles. One platform.

01

Super Admin

Creates and manages Admin accounts. Can block admins, view analytics, and oversee the entire system.

POST /superadmin/create-admin PUT /superadmin/block-admin/{id}
→
02

Admin

Creates interview slots with title, date/time, and duration. Manages the pool of available booking slots.

POST /admin/create-slot GET /slots
→
03

User

Browses available slots, books interviews with one click, and views their booking history anytime.

GET /slots POST /slots/book/{id}
